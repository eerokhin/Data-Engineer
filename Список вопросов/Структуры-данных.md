# Cтруктура данных

Структура данных — это способ хранения данных.

Например:

- список (`list`) хранит элементы подряд;
- словарь (`dict`) хранит пары ключ-значение;
- множество (`set`) хранит уникальные элементы;
- стек хранит элементы по принципу последний пришёл — первый ушёл;
- очередь хранит элементы по принципу первый пришёл — первый ушёл.

## Stack (Стек)

Stack (стек) — это базова структура данных, которая управляет порядком обработки элементов. Работает по принципу: `LIFO — Last In, First Out`. Последний положили → первый достали.

В Python нет отдельного встроенного типа `Stack`. Поэтому стек обычно реализуют через список (`list`). А у списка уже свои методы. `stack = []`

**Основные операции**

| В теории (Stack)   | Python            | Что это значит простыми словами               |
| ------------------ | ----------------- | --------------------------------------------- |
| `push(x)`          | `stack.append(x)` | **Положить элемент наверх стека. Потому что `append()` добавляет элемент в конец списка, а конец списка мы считаем верхушкой стека.**             |
| `pop()`            | `stack.pop()`     | **Убрать и вернуть верхний элемент**          |
| `top()` / `peek()` | `stack[-1]`       | **Посмотреть верхний элемент, не удаляя его** |
| `isEmpty()`        | `if not stack:`   | **Проверить, пуст ли стек**                   |
| `size()`           | `len(stack)`      | **Сколько элементов сейчас в стеке**          |
