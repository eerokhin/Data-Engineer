# Cтруктура данных

Структура данных — это способ хранения данных.

Например:

- список (`list`) хранит элементы подряд;
- словарь (`dict`) хранит пары ключ-значение;
- множество (`set`) хранит уникальные элементы;
- стек хранит элементы по принципу последний пришёл — первый ушёл;
- очередь хранит элементы по принципу первый пришёл — первый ушёл.

## Stack (Стек)

Stack (стек) — это базова структура данных, которая управляет порядком обработки элементов. Работает по принципу: `LIFO — Last In, First Out`. Последний положили → первый достали.

В Python нет отдельного встроенного типа `Stack`. Поэтому стек обычно реализуют через список (`list`). А у списка уже свои методы. `stack = []`

**Основные операции**

| В теории (Stack)    | Python            | Что это значит простыми словами               |
| ------------------- | ----------------- | --------------------------------------------- |
| `push(x)`           | `stack.append(x)` | **Положить элемент наверх стека. Потому что `append()` добавляет элемент в конец списка, а конец списка мы считаем верхушкой стека.**             |
| `pop()`             | `stack.pop()`     | **Убрать и вернуть верхний элемент**          |
| `top()` / `peek()`  | `stack[-1]`       | **Посмотреть верхний элемент, не удаляя его** |
| `isEmpty()`         | `if not stack:`   | **Проверить, пуст ли стек**                   |
| `size()`            | `len(stack)`      | **Сколько элементов сейчас в стеке**          |

## Queue - Очередь

Очередь — структура данных, работающая по принципу: `FIFO — First In, First Out`. Первый вошёл — первый вышел. Как очередь в магазине.

В Python очередь (Queue) реализуется двумя основными способами, но на практике почти всегда используют `collections.deque`. Список `list` использовать не рекомендуется.

```python
from collections import deque
q = deque()
```

| Название операции  | Как называется   | Команда в Python          | Что делает                               |
| ------------------ | ---------------- | ------------------------- | ---------------------------------------- |
| Добавить в очередь | **enqueue**      | `q.append(x)`             | Добавляет элемент **в конец** очереди    |
| Удалить из очереди | **dequeue**      | `q.popleft()`             | Удаляет и возвращает **первый** элемент  |
| Посмотреть первый  | **peek / front** | `q[0]`                    | Показывает первый элемент, **не удаляя** |
| Проверить пустоту  | **isEmpty**      | `len(q) == 0` или `not q` | Возвращает `True`, если очередь пустая   |
| Размер очереди     | **size**         | `len(q)`                  | Количество элементов в очереди           |

Различие между `list` и `deque` в Python

`list` — это динамический массив, удобный для доступа по индексу.

Удаление элемента из начала списка (`pop(0)`) работает за `O(n)`, потому что все элементы сдвигаются.

`list` плохо подходит для реализации очереди с удалением элементов слева.

`deque (collections.deque)` — это двусторонняя очередь.

`deque` поддерживает добавление и удаление элементов с обоих концов за `O(1)`.

Метод `popleft()` в `deque` удаляет первый элемент без сдвига остальных.

`deque` лучше всего подходит для задач типа «скользящее окно» (sliding window).

Для учебных примеров можно использовать `list`, но в production и на собеседовании предпочтительнее `deque`.

```python
from collections import deque

q = deque()

q.append(5)        # enqueue
q.append(10)

first = q[0]       # peek
removed = q.popleft()  # dequeue

empty = not q      # isEmpty
```
