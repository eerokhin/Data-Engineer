**1. Что такое переменная в Python?**

Переменная — это имя, которое ссылается на объект в памяти.

`x = 10`

`x` хранит не само число, а ссылку на объект `10` в памяти.

```python

print(id(a), id(b))# -> 134703985937344 134703985937344

a = 5
b =  a
print(id(a), id(b)) # -> 11645480 11645480

print(a, b) # -> 5 6
```

**2. Что такое объект в Python?**

Практически всё в Python — объект: числа, строки, списки, функции, классы

У объекта есть: тип, значение, адрес в памяти

**3. Чем отличается mutable и immutable?**

Immutable — неизменяемые. Нельзя изменить после создания.

Примеры: 
Числа(`int`, `float`, `complex`), Строки(`str`), Кортежи(`tuple`), Логические(`bool`)

```python
a = 5
print(id(a)) # Адрес 11645480
a += 1
print(id(a)) # АДРЕС ИЗМЕНИЛСЯ! 11645512
```

Создается новый объект 6, адрес изменился.

Mutable — изменяемые. Можно менять объект в памяти.

Примеры: Списки(`list`), Словари(`dict`), Множества(`set`)

```python
lst = [1]
print(id(lst)) # Адрес 134703988240704
lst.append(2)
print(id(lst)) # Адрес тот же 134703988240704
```

Список изменяется на месте.

**6. Чем отличается `==` от `is`?**

`==` Сравнивает значения.

`is` Проверяет, один ли это объект в памяти.

```python
a = [1]
b = [1]
print(id(a), id(b)) # -> 134704683147456 134703986448128
print(a == b) # -> True
print(a is b) # -> False

a = 5
b = 5
print(id(a), id(b)) # -> 11645480 11645480
print(a == b) # -> True
print(a is b) # -> True
```

У чисел одинаковый `id`, так как Python не создает два объекта `5`. Он использует один общий объект из кэша.

Python так делает из за оптимизаии памяти и скорости. Числа, используются очень часто они immutable, безопасны для переиспользования. Python обычно кэширует: -5 ... 256.
Списки нельзя кэшировать как числа. Потому что списки mutable. Если бы Python переиспользовал один список: `a = [1]` и `b = [1]` и это был бы один объект, то: `a.append(2)` сломало бы `b`.
Immutable объекты можно безопасно переиспользовать. Mutable объекты нельзя безопасно шарить между переменными автоматически.


Строки тоже иногда имеют одинаковый id:

```python
a = "hello"
b = "hello"
print(a is b)
Часто: True
```

Потому что строки тоже могут интернироваться (кэшироваться). Но не всегда.

```python
a = "hello world this is long"
b = "hello world this is long"
```

`is` может быть как `True`, так и `False` — зависит от оптимизаций Python. Поэтому никогда не сравнивают строки через `is`
Правильно: `a == b`

Кэш — это временное хранилище данных, чтобы не вычислять или не загружать что-то повторно. Главная цель кэша: ускорение работы; экономия ресурсов.

**7. Что такое протокол итерации Python?**

Протокол итерации — это механизм, через который Python проходит по объектам в цикле `for`.

Когда Python видит:

```python
for x in obj:
```

он внутри вызывает:

```python
it = iter(obj) # вызывает obj.__iter__() и возвращает итератор.
#obj — это итерируемый объект (iterable), то есть объект, из которого можно получить итератор.
#it — это итератор (iterator), который хранит текущее состояние и возвращает элементы по одному через next().
```

то есть метод `__iter__()`.

После этого Python начинает получать элементы по одному через:

```python
next(it)
```

то есть через метод `__next__()`.

Когда элементы заканчиваются, вызывается исключение `StopIteration`, и цикл завершается.


```python
obj = [1, 2, 3]
it = iter(obj)
next(it)  # 1
next(it)  # 2
next(it)  # 3
```

**8. Что такое итератор?**

Итератор — это объект, который умеет выдавать элементы по одному и хранит состояние обхода.

У него есть метод `__next__()`, который возвращает следующий элемент.

Например, после вызова `iter(list)` Python создаёт специальный объект-итератор, который помнит текущую позицию в списке.

**9. Какие магические методы нужны для реализации итератора?**

Для реализации собственного итератора нужно реализовать два магических метода:

1. `__iter__()` — должен вернуть объект-итератор
2. `__next__()` — должен вернуть следующий элемент или вызвать `StopIteration`, если элементы закончились

**10. Чем итератор отличается от генератора?**

Генератор — это упрощённый способ создания итератора.

Итератор обычно пишется через класс и вручную реализует `__iter__()` и `__next__()`.

Генератор создаётся через функцию с `yield`, а Python автоматически делает из неё итератор.

То есть каждый генератор является итератором, но не каждый итератор — генератор.

**11. Как написать свой кастомный итератор?**

Нужно создать объект, который: можно использовать в `for`, умеет выдавать элементы по одному, помнит своё состояние.

Для этого нужны 2 метода:

`__iter__()` -	возвращает итератор

`__next__()` -	возвращает следующий элемент


Самый простой кастомный итератор который считает от 1 до 5.

```python
class Counter:
    def __init__(self, max_value):
        self.max_value = max_value # до какого числа считаем
        self.current = 0 # текущее состояние итератора

    def __iter__(self):
        #Python вызывает этот метод, когда делает: iter(counter)
        return self # Почему return self. Потому что сам объект уже является итератором.

    def __next__(self):
        #Этот метод вызывается каждый раз при: next(iterator)
        if self.current < self.max_value:
            self.current += 1
            return self.current

        raise StopIteration

counter = Counter(5)
for x in counter:
    print(x)
```

**12. Что такое генератор?**

Генератор в Python — это объект-итератор, который создаёт значения по одному с помощью `yield`.

После `yield` функция не завершается, а ставится на паузу и продолжает выполнение при следующем вызове `next()`.

Генераторы позволяют экономить память, потому что не хранят все данные сразу.

**14. Разница iterable и iterator?**

*Iterable* - Объект, по которому можно пройтись в `for`.

Примеры: `list`, `tuple`, `str`, `dict`, `set`.

Можно преобразовать в iterator через `iter()`.

*Iterator* - Объект, который уже “идёт по элементам”.

- имеет `__next__()`
- хранит текущее состояние

**15. Как работает yield?**

`yield` останавливает выполнение функции и возвращает значение, сохраняя её состояние. При следующем вызове `next()` функция продолжает работу с места остановки.

**16. Что такое lazy evaluation?**

Lazy evaluation в Python реализуется через генераторы: значения не вычисляются заранее, а создаются только при обращении через `next()` или цикл `for`. 
Это позволяет экономить память и работать с большими данными.

**17. Почему генераторы экономят память?**

Генераторы экономят память, потому что они не создают и не хранят всю последовательность в памяти. Вместо этого они вычисляют каждое значение по одному при помощи `yield`, сохраняя только состояние выполнения функции.

**19. Можно ли пройтись по генератору дважды?**

Нет

Генератор нельзя пройти дважды, потому что он является одноразовым итератором. Он не хранит значения, а генерирует их на лету и “исчерпывается” после первого прохода. Чтобы пройтись снова, нужно создать новый генератор или сохранить данные в структуру вроде списка.

Генератор — это одноразовый iterator, который:

- выдаёт значения через `yield`
- запоминает своё текущее состояние
- “сгорает” после завершения

После того как значения закончились, он становится пустым.

Пример

```python
def gen():
    yield 1
    yield 2
    yield 3

g = gen()

for x in g:
    print(x)

for x in g:
    print(x)
```

Результат:

```
1
2
3
# второй цикл ничего не выведет
```

**20. Чем yield отличается от return?**

`return` завершает функцию и возвращает значение, а `yield` превращает функцию в генератор, который возвращает значения по одному и приостанавливает выполнение, сохраняя своё состояние для продолжения.

**21. Почему генераторы быстрее/лучше?**

Генераторы лучше не из-за скорости вычислений, а из-за экономии памяти и ленивых вычислений. Они создают значения по одному через `yield`, что позволяет обрабатывать большие данные без загрузки всей последовательности в память.

**22. Что возвращает функция с yield?**

Функция с `yield` возвращает объект генератора (generator object), который не вычисляет значения сразу, а выдаёт их по одному при вызове `next()` или в цикле `for`.

**23. Назовите основные принципы ООП?**

Объектно-ориентированный подход к программированию строится на трёх основных принципах: наследование, инкапсуляция и полиморфизм.

*Инкапсуляция* — это принцип ООП, при котором внутренние данные объекта скрыты и доступны только через контролируемый интерфейс (методы).

```python
class BankAccount:
    def __init__(self, balance):
        self.__balance = balance  # скрытое поле

    def deposit(self, amount):
        self.__balance += amount

    def get_balance(self):
        return self.__balance

acc = BankAccount(100)
acc.deposit(50)

print(acc.get_balance())  # 150
```

Суть:
- `__balance` нельзя менять напрямую
- доступ только через методы (`deposit`, `get_balance`)
- это и есть инкапсуляция: данные защищены + контролируемый доступ

*Наследование* — возможность создавать новые классы на основе существующих и переиспользовать код

*Полиморфизм* — это когда один и тот же метод вызывается у разных объектов, но каждый объект реализует его по-своему.

```python
class Dog:
    def speak(self):
        return "Гав"

class Cat:
    def speak(self):
        return "Мяу"

animals = [Dog(), Cat()]

for animal in animals:
    print(animal.speak())
```

Что происходит

Один и тот же вызов: `animal.speak()` но результат разный:

```python
Dog → "Гав"
Cat → "Мяу"
```

Один метод — разное поведение в зависимости от объекта.

**26. Лямбда функция (что это, зачем, где использовать)?**

Lambda-функция — это анонимная функция в Python, которая используется для простых однострочных выражений. Чаще всего её применяют в функциях высшего порядка, таких как `sorted`, `map`, `filter` или `apply` в pandas, когда нужно быстро передать небольшую логику без объявления отдельной функции через `def`. Если логика сложная, лучше использовать обычную функцию для читаемости.

Пример:

Простое увеличение числа

```python
f = lambda x: x + 1
print(f(5))  # 6
```

`sorted()` по возрасту

```python
users = [
        {"name": "Anna", "age": 30},
        {"name": "Ivan", "age": 25}
        ]
sorted_users = sorted(users, key=lambda x: x["age"])
print(sorted_users) # [{'name': 'Ivan', 'age': 25}, {'name': 'Anna', 'age': 30}]
```

`filter()` — только чётные числа

```python
nums = [1, 2, 3, 4, 5, 6]

even = list(filter(lambda x: x % 2 == 0, nums))
print(even)  # [2, 4, 6]
```

`map()` — возвести в квадрат

```python
nums = [1, 2, 3, 4]

squares = list(map(lambda x: x ** 2, nums))
print(squares)  # [1, 4, 9, 16]
```

pandas `apply`

```python
import pandas as pd
df = pd.DataFrame({"salary": [1000, 2000, 3000]})
df["salary_k"] = df["salary"].apply(lambda x: x / 1000)
print(df)

#     salary  salary_k
#0    1000       1.0
#1    2000       2.0
#2    3000       3.0
```

**27. В чем разница между определением функции `func` и `func()`?**

`func` — это объект функции, а `func()` — это вызов функции.

`func`: Это объект самой функции, который можно сохранить в переменную, передать в качестве аргумента другой функции. 
Например, если присвоить `func` новой переменной, например, `new_func` = `func`, то `new_func()` будет эквивалентен вызову `func()`. 
Без круглых скобок функция не выполняется, она просто существует как объект.

`func()`: Это фактический вызов функции. Круглые скобки указывают Python на выполнение кода внутри функции и возврат её результата. 
Если бы функция принимала аргументы, они передавались бы в скобках — например, `func(arg1, arg2)`.

Пример для наглядности:

```python
def func():
  print('I am a function')
```

```python
# Присваиваем функцию переменной
new_func = func
new_func() # Вывод: I am a function
```

```python
# Передаем функцию как аргумент другой функции
def call_func(f):
  f() # Вызов переданной функции
  
call_func(func) # Вывод: I am a function
```

**29. Как рассчитывается сложность алгоритма? На примере `list`, `tuple`**

Сложность алгоритма оценивается через `Big-O` нотацию — это способ описать, как растёт время выполнения или память при увеличении размера входных данных `(n)`.

Сложность основных операций:

```python
a[i] #список
t[i] #кортеж
```

Доступ по индексу - `O(1)` для списка и кортежа, элементы лежат в памяти подряд

Поиск элемента (`x in a`, `x in t`) - `O(n)` для списка и кортежа, нужно проверить каждый элемент последовательно

Добавление элемента - `O(1)` для списка, кортеж нельзя изменить, любое добавление = создание нового объекта поэтому `O(n)` создаётся новый tuple, копируются все элементы

Изменение элемента - `O(1)` для списка, tuple невозможно изменить, неизменяемый

Удаление 

- `list`: `pop()` с конца → `O(1)`
- удаление из середины → `O(n)`
- `tuple`: невозможно → только пересоздание → `O(n)`

Сложность алгоритма оценивается через `Big-O` нотацию, которая показывает рост количества операций при увеличении входных данных.
Например, у `list` и `tuple` доступ по индексу одинаковый — `O(1)`, потому что данные лежат в памяти последовательно.
Но `list` является изменяемым, поэтому операции вставки и удаления могут быть `O(1)` амортизированно или `O(n)`, тогда как `tuple` — неизменяемый, и любые изменения требуют создания нового объекта, что делает такие операции `O(n)`. Поэтому выбор структуры данных напрямую влияет на сложность алгоритма.

**32. Что такое итерация?**

Итерация — это один шаг выполнения цикла, при котором обрабатывается один элемент последовательности. Например, если цикл `for` проходит по списку из пяти элементов, то он выполнит пять итераций. 
Внутри Python цикл `for` работает через протокол итерации: сначала получает итератор с помощью `iter()`, затем последовательно вызывает `next()`, пока итератор не сообщит о завершении с помощью исключения `StopIteration`.

Проще говоря, итерация — это один проход цикла.

Например, если у нас есть список

`numbers = [10, 20, 30]`

то цикл

```python
for number in numbers:
    print(number)
```

выполнит три итерации:

```
Итерация	Значение number
1	        10
2	        20
3	        30
```

После третьей итерации элементы заканчиваются, и цикл завершается.

Важно помнить:

Итерация — один шаг обхода элементов.

Цикл — конструкция, выполняющая множество итераций.

Итератор — объект, который выдает элементы по одному.

**33. Можно ли на одну функцию нацепить несколько декораторов и как они будут считываться?**

Да, одну функцию можно обернуть несколькими декораторами. Python применяет их снизу вверх.

```python
@A
@B
def func():
    ...
```

эквивалентна:

```python
func = A(B(func))
```

Однако при вызове функции выполнение начинается с внешнего декоратора `(A)`, затем переходит к внутреннему `(B)`, затем вызывается сама функция, после чего управление возвращается обратно в обратном порядке. Это происходит потому, что каждый декоратор возвращает новую функцию-обёртку `(wrapper)`, которая становится входом для следующего декоратора.

- Применение декораторов (оборачивание функции): снизу вверх.

- Выполнение при вызове функции: сверху вниз, затем обратно вверх.

<details>
    
<summary>Пример декоратора</summary>

```python
def decorator1(func):
    def wrapper():
        print("До decorator1")
        func()
        print("После decorator1")
    return wrapper


def decorator2(func):
    def wrapper():
        print("До decorator2")
        func()
        print("После decorator2")
    return wrapper


@decorator1
@decorator2
def hello():
    print("Hello")
hello()

# До decorator1
# До decorator2
# Hello
# После decorator2
# После decorator1
```

Эквивалентно записи: `hello = decorator1(decorator2(hello))`

При вызове:

- сначала заходим во внешний слой (decorator1);
- затем во внутренний (decorator2);
- затем вызывается сама функция;
- после этого выполнение выходит обратно наружу.

</details>

**35. Что представляет из себя тип данных Int в Python?**

`int` — это встроенный неизменяемый тип данных Python для хранения целых чисел.

**36. Какая типизация используется в Python?**

Python использует динамическую и сильную типизацию. Динамическая типизация означает, что тип переменной определяется во время выполнения и может изменяться. Сильная типизация означает, что Python не выполняет неявные преобразования несовместимых типов — их нужно делать явно. Например, строку нельзя сложить с числом без приведения типов.

<details>
    
<summary>Динамическая типизация</summary>

Это значит: тип переменной определяется во время выполнения программы, а не заранее.

Пример:

```python
x = 10
x = "hello"
x = [1, 2, 3]
```

Одна и та же переменная может хранить разные типы данных.

Python не требует заранее объявлять тип: `int x;  # как в Java — в Python так нельзя`

</details>

<details>
    
<summary>Сильная типизация</summary>

Это значит: Python не делает автоматические небезопасные преобразования типов.

Пример:

```python

```

Потому что Python не будет сам складывать строку и число.

Нужно явно привести тип: `int("10") + 5`

</details>

**37. Как происходит поиск в set и списке?**

Поиск элемента в `list` и `set`

Поиск в `list`

При проверке:

```python
x in my_list
```

Python перебирает элементы по очереди, пока не найдет нужный или не дойдет до конца списка.

* Средняя и худшая сложность: **O(n)**.
* Чем больше список, тем дольше поиск.

Поиск в `set`

При проверке:

```python
x in my_set
```

Python использует хеш-таблицу:

1. Вычисляет хеш объекта (`hash(x)`).
2. По хешу определяет, где элемент должен находиться.
3. Проверяет только эту позицию.

* Средняя сложность: `O(1)`.
* Худшая сложность (из-за коллизий): `O(n)`, но на практике встречается редко.

Когда использовать:

* `list` — если важен порядок элементов или нужен доступ по индексу.
* `set` — если требуется часто проверять наличие элемента или удалять дубликаты.

Пример

```python
numbers_list = [1, 2, 3, 4, 5]
numbers_set = {1, 2, 3, 4, 5}

print(3 in numbers_list)  # O(n)
print(3 in numbers_set)   # O(1)
```

Вывод: если основная операция — поиск элемента, `set` значительно эффективнее `list`.

**38. Что может быть ключём словаря?**

Ключом словаря может быть любой неизменяемый (hashable) объект.

- `bool`
- `None`

Примеры

```python
d = {
    "name": "Ivan",

    (1, 2): "point",

}
```

Какие типы нельзя использовать:`list`, `dict`, `set`

Словарь в Python — это хеш-таблица. Он работает так: `ключ → hash() → индекс в памяти → значение`. Если объект изменяемый (например `list`), его `hash` может измениться → словарь перестанет находить значение корректно.

**40. Что такое `try`, `except`, `else`, `finally`?**

`try / except / else / finally` — это конструкция в Python для обработки ошибок (исключений), чтобы программа не “падала” при сбое.

- `try` — код, где может быть ошибка
- `except` — обработка ошибки
- `else` — выполняется, если ошибки не было
- `finally` — выполняется всегда (например, для закрытия ресурсов)

```python
try:
    file = open("data.txt")
    data = file.read()
except FileNotFoundError:
    print("Файл не найден")
else:
    print("Файл прочитан успешно")
finally:
    print("Закрываем ресурсы")

#Файл не найден
#Закрываем ресурсы
```

```
try → пробуем код
    ↓
ошибка?
   ├── да → except
   └── нет → else
    ↓
finally → всегда выполняется
```

**41. Что выведет `print(a)`?**

```python

```

Как работает:

- `a` создаётся список в памяти
- `b` не копия, а просто ещё одна ссылка на тот же объект
- Теперь: `a`→ тот же список и `b` → тот же список
- `b.append(4)` мы изменяем сам объект списка, а не переменную
- Итог: и `a`, и `b` смотрят на один и тот же список: `[1, 2, 3, 4]`

В Python переменные не хранят объекты — они хранят ссылки на объекты.
Поэтому `b = a` не создаёт копию, а делает вторую ссылку на тот же объект.

Если нужно именно копию:

```python
b = a.copy()
# или
b = a[:]
```

**42. Как работает сборщик мусора в python?**

В Python основной механизм очистки памяти — это подсчёт ссылок (Reference counting): объект удаляется, когда на него больше нет ссылок. Но этот механизм не обрабатывает циклические ссылки, поэтому используется дополнительный сборщик мусора (Garbage collector (GC — сборщик циклов)), который периодически ищет и удаляет такие циклы.

*Reference counting*

Ссылка исчезает в 3 случаях:
1. Перезапись переменной
   ```
   a = [1, 2, 3]
   a = 10
   ```
2. Удаление переменной
   ```
   a = [1, 2, 3]
   del a
   ```
3. Выход из области видимости (scope)
   ```
   def f():
       x = [1, 2, 3]
   f()
   #после выполнения функции: x уничтожается, ссылка исчезает
   ```
Как счётчик становится 0?

У каждого объекта есть счётчик.

```
a = [1,2,3]   # 1 ссылка
b = a         # 2 ссылки
```

Убираем одну ссылку: `del a`. Теперь: остаётся только `b`, счётчик = 1. Убираем последнюю: `del b`. Теперь: никто не держит объект, счётчик = 0. Когда счётчик становится `0` → Python сразу удаляет объект.

*GC (Garbage Collector)*

```
a = []
b = []

a.append(b)
b.append(a)
```

Получился “замкнутый круг”: `a` держит `b`, `b` держит `a`

Теперь: `del a`, `del b`. Но объекты всё ещё держат друг друга. Счётчик ссылок НЕ становится 0, значит они НЕ удаляются. 

GC ищет группы объектов, которые ссылаются друг на друга, проверяет можно ли до них добраться извне(“Можно ли до тебя добраться из программы?”), если нельзя то удаляет их.

GC в Python использует поколенческую модель: объекты делятся на 3 поколения. Новые объекты попадают в Gen 0 и проверяются чаще всего. Если они “выживают”, то переходят в более старшие поколения, которые проверяются реже. Это оптимизация, основанная на том, что большинство объектов живёт недолго.

- Generation 0 — новорождённые, только что созданные объекты, живут секунды/миллисекунды, их очень много, Python проверяет их ЧАСТО. GC чистит Gen 0, если объект мусор — удаляет, если нет — переводит в Gen 1.
- Generation 1 — подростки, пережили первую проверку, уже “чуть более важные”, проверяются реже. Если пережил ещё — идёт в Gen 2. 
- Generation 2 — долгоживущие, живут долго, обычно это: кеши, глобальные структуры, важные данные. Проверяются очень редко. Gen 2 почти не трогают. Потому что: “если объект живёт долго — скорее всего он нужен”.
