# Что такое генератор в Python

Генератор (`generator`) — это объект-итератор, который отдаёт значения по одному.

Главная идея генератора:

- значения создаются только тогда, когда они нужны
- все данные не хранятся в памяти сразу
- генераторы помогают экономить память

---

# Обычная функция

Обычная функция завершает работу после `return` и возвращает все данные.

```python
def numbers():
    return [1, 2, 3]
```

Использование:

```python
nums = numbers()

print(nums)
```

Результат:

```python
[1, 2, 3]
```

Весь список создаётся сразу в памяти.

---

# Генератор

Генератор создаётся с помощью `yield`.

```python
def numbers():
    yield 1
    yield 2
    yield 3
```

---

# Что возвращает генератор

```python
gen = numbers()

print(gen)
```

Результат:

```python
<generator object numbers at 0x...>
```

Функция с `yield` возвращает объект генератора.

---

# Как получать значения

Генератор отдаёт значения по одному через `next()`.

```python
print(next(gen))
print(next(gen))
print(next(gen))
```

Результат:

```python
1
2
3
```

После окончания значений:

```python
next(gen)
```

получим:

```python
StopIteration
```

---

# Как работает yield

Когда Python встречает `yield`:

1. возвращается значение
2. функция ставится на паузу
3. состояние функции сохраняется
4. следующий `next()` продолжает выполнение с места остановки

---

# Пример работы генератора

```python
def test():
    print("A")
    yield 1

    print("B")
    yield 2


g = test()

print(next(g))
print(next(g))
```

Результат:

```python
A
1
B
2
```

Что произошло:

## Первый next(g)

Функция:

- запустилась
- вывела `"A"`
- дошла до `yield 1`
- вернула `1`
- остановилась

## Второй next(g)

Функция:

- продолжилась с места остановки
- вывела `"B"`
- вернула `2`

---

# Отличие yield от return

## return

```python
def f():
    return 1
```

- полностью завершает функцию
- возвращает одно значение

---

## yield

```python
def f():
    yield 1
```

- не завершает функцию
- ставит выполнение на паузу
- позволяет получать значения по одному

---

# Генератор является итератором

Любой генератор:

- поддерживает протокол итерации
- имеет методы:
  - `__iter__()`
  - `__next__()`

Поэтому генераторы можно использовать в цикле `for`.

---

# Пример с for

```python
def counter():
    for i in range(5):
        yield i


for x in counter():
    print(x)
```

Результат:

```python
0
1
2
3
4
```

---

# Генераторное выражение

## Список

```python
nums = [x for x in range(5)]
```

## Генератор

```python
nums = (x for x in range(5))
```

Разница:

- `[]` создаёт список
- `()` создаёт генератор

---

# Зачем нужны генераторы

Генераторы полезны при работе с большими объёмами данных.

Например:

- большие файлы
- логи
- API
- streaming данных
- ETL-пайплайны

Позволяют:

- Снижать нагрузку на память. Генераторы помогают обрабатывать большие объёмы данных, например считывать файлы построчно без загрузки в оперативную память.
-  Работать с бесконечными последовательностями. Генераторы создают потоки данных, например случайные числа или значения в цикле без предела.
- Оптимизировать код. Они делают код компактнее и удобнее: не нужно использовать дополнительные структуры для хранения промежуточных данных.

---

# Преимущество генераторов

## Без генератора

```python
nums = [x for x in range(1000000)]
```

Создаётся большой список в памяти.

---

## С генератором


Значения создаются по одному по мере необходимости.

# Кратко

Генератор в Python — это объект-итератор, который создаёт значения по одному с помощью `yield`.

После `yield` функция не завершается, а ставится на паузу и продолжает выполнение при следующем вызове `next()`.

Генераторы позволяют экономить память, потому что не хранят все данные сразу.

---

# Практическое применение генераторов в Python

Генераторы особенно полезны при работе с:

- большими файлами
- потоковыми данными
- бесконечными последовательностями
- API
- генерацией данных “на лету”

Главная идея — **не загружать всё в память сразу**.

---

# 1. Чтение больших файлов построчно

Если файл большой, загружать его целиком в память — плохо.

Генератор позволяет читать построчно:

```python
def read_large_file(filename):
    with open(filename, "r", encoding="utf-8") as file:
        for line in file:
            yield line.strip()
```

Использование:

```python
for line in read_large_file("huge_data.txt"):
    print(line)
```

- строки обрабатываются по одной  
- память не перегружается

---

# 2. Бесконечные последовательности

Генераторы могут создавать бесконечные потоки данных.

Пример — генератор ID:

```python
def unique_id_generator():
    id = 1
    while True:
        yield id
        id += 1
```

Использование:

```python
gen = unique_id_generator()

print(next(gen))  # 1
print(next(gen))  # 2
print(next(gen))  # 3
```

- значения генерируются бесконечно  
- ничего не хранится в памяти

---

# 3. Генерация случайных данных

Можно использовать генератор для паролей:

```python
import random
import string

def random_password(length=8):
    while True:
        yield ''.join(
            random.choices(string.ascii_letters + string.digits, k=length)
        )
```

Использование:

```python
password_gen = random_password()

print(next(password_gen))
print(next(password_gen))
```

- каждый вызов = новый пароль  
- нет хранения списка паролей

---

# 4. Потоковая обработка данных из API

Генераторы удобно использовать при работе с API, чтобы обрабатывать данные постепенно.

```python
import requests

def fetch_data(url):
    response = requests.get(url, stream=True)
    for line in response.iter_lines():
        if line:
            yield line.decode("utf-8")
```

Использование:

```python
for data in fetch_data("https://example.com/large-data"):
    print(data)
```

- данные читаются по мере поступления  
- не загружается весь ответ целиком
